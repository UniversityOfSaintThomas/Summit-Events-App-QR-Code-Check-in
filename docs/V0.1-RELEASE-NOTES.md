## ðŸ’¡ User Guide

### Starting a Check-In Session

1. **Select Event Date** - Choose the date of your event
2. **Select Event Instance** - Pick the specific instance from dropdown
3. **Click "Start Scanning Session"** - Activates the check-in interface

### Checking In Registrants

#### Method 1: Camera Scan (Recommended)
1. Click **"Scan with Camera"**
2. Grant camera permissions (first time only)
3. Point camera at QR code on registration badge
4. Review registration details displayed
5. Click **"Check In"** to confirm
6. Repeat for next registrant

#### Method 2: Manual Search
1. Enter **First Name**, **Last Name**, or **Email**
2. Click **"Search"**
3. Select registrant from results list
4. Review registration details
5. Click **"Check In"** to confirm

### Undo Accidental Check-In

If you accidentally check in the wrong person:
1. Scan or search for the registration again
2. Notice the "Already Checked In" badge
3. Click **"Undo Check-In"** button
4. Status reverts to "Registered"
5. Counter decrements

### Ending a Session

**Option 1: End Session**
- Session menu â†’ End Session
- Shows summary (checked in X registrants in Y minutes)

**Option 2: Reset Counter**
- Session menu â†’ Reset Counter
- Clears session count but keeps session active

## ðŸ› ï¸ Troubleshooting

### Camera Not Working

**Problem:** Camera doesn't activate when clicking "Scan with Camera"

**Solutions:**
1. **Check Browser Permissions:**
   - Chrome: Address bar â†’ Camera icon â†’ Allow
   - Firefox: Address bar â†’ Lock icon â†’ Permissions â†’ Camera â†’ Allow
   - Safari: Safari menu â†’ Settings â†’ Websites â†’ Camera â†’ Allow

2. **Check Browser Support:**
   - Must use Chrome 83+, Firefox 78+, Edge 83+, or Safari 14+
   - Private/Incognito mode may block camera

3. **Device Detection:**
   - Salesforce Mobile App: Uses native scanner automatically
   - Desktop/Browser: Uses jsQR library
   - Check console for errors

### "No Registration Found" Error

**Problem:** Scanning QR code returns "No registration found"

**Solutions:**
1. Verify QR code contains valid registration ID
2. Check registration belongs to selected event instance
3. Confirm registration status is not "Cancelled"
4. Verify user has Read permission on Registration object

### Permission Errors

**Problem:** "Insufficient Privileges" or "You do not have access"

**Solutions:**
1. Enable `summitEventsCheckin` Apex class for profile
2. Grant Read/Edit on `Summit Events Registration` object
3. Grant Read on `Summit Events Instance` and `Summit Events` objects
4. Check field-level security on Status field

### Search Returns No Results

**Problem:** Manual search doesn't find registrations

**Solutions:**
1. Verify event instance is selected
2. Check spelling of names
3. Try partial name (e.g., "John" instead of "Jonathan")
4. Try email search instead
5. Verify registrations exist for selected instance
